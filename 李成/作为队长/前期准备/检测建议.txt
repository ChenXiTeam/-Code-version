
// pls用记事本40列窗口查看

    读了你们的项目doc，我按照我的理解，阐述我的观点。注意，可能有故意的偏差，这些偏差代表了我的指导倾向。

    该项目针对软件工程专业学生作业和/或实验报告，为了减轻教师批改工作量，提供效率，项目设定功能可能包括但不仅限于：
（1）检查代码，能自动发现可能的语法错、编译错、运行错等不同级别的错误；
（2）检查文字，能发现抄袭自网络还是抄袭自同学，如果有雷同抄袭的话。

    代码的提取：
    （1）可以直接要求学生提交源代码，即.c或.java或.py等格式源文件；
    （2）可以规定学生嵌在作业中的代码前后有标记，像这样：
------begin-java-code------
public class HelloWorld {
	public static void main(String[] args) {
		System.out.println("Hello World!");
	}
}
------end-java-code------
    （3）可以从作业中的代码截屏图片中提取代码。

    检查代码：
    （1）静态语法检查，比如C语言代码可以用lint，Java代码用SonarLint等
    （2）尝试调用外部编译器编译一下
    （3）可以运行一下，看看结果是否符合预期，如果编译能通过的话

    提取作业正文文字：
    （1）支持doc、pdf、md、html、wps、rtf等等各种常见格式文本
    （2）使用apache tika工具提取
    （3）要求学生作业必须支持tika，比如不能对pdf使用口令保护。

    发现雷同或抄袭(比较相似程序)：
    （1）比较代码的相似度和文字的相似度，方法可能接近；
    （2）比较两段文字的相似度，应该有成熟的算法或工具，自己找找看。
    （3）比较两段代码的相似度，应该有成熟的算法或工具
    （4）查询一段文字是否是抄袭（抄袭自某个文档库），应该有成熟的算法或工具

    总之，把这个项目部分功能简单实现，应该是容易的，但是都实现应该很难。尽量做，希望有好成果。
    以上仅供参考，并不是要求或承诺能实现。

    找技术创新点的话，参照你发给我的项目文档，可以这么写：
    （1）支持多种文档格式doc、xps、pdf、html、md等。
    （2）能识别图片中的代码（或片段）。
    （3）能检查代码的各种层次的错误：静态语法错、编译错、运行错等。
    （4）也能提取正文。
    （5）能比较代码或正文的相似度。


一些比较简单的	（任务的本身重复率就高，没必要查重，只检查正确性即可）
一些比较复杂的	（任务本身重复率低，有必要查重，正确性也有必要）
